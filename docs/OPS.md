# 操作说明 · 进销存

> 公开版 · 不含环境专属地址与密钥

## 0. 职责拆分

| 场景 | 用什么 | 不做什么 |
|------|--------|----------|
| **单行日常领用/消耗** | 多维表格「站库存」上的原生「领用」**按钮字段** + 自动化 | 不要靠插件列表行内小按钮当主入口 |
| **多行一次领用** | 边栏插件「批量领用」：表内勾选 → 读取勾选 → 二次确认过账 | 不要在插件里仿造整张台账当主操作面 |
| 直接改「现存量」 | 仅盘点/纠错（并写修改记录） | 禁止改数后系统猜「是不是领用」 |

对齐常见做法：复杂批量在边栏插件；单行快捷用表格按钮/自动化。

---

## 1. 原生「领用」按钮（在表格 UI 配置）

按钮字段通常**不能**靠 OpenAPI 稳定创建，请在表格中点「+」加列。

### 1.1 推荐字段（站库存）

| 字段 | 类型 | 用途 |
|------|------|------|
| 待领数量 | 数字 | 点按钮前填本次数量；过账后清零 |
| 待领用人 | 文本 | 可选；写入出库流水「领用人」 |
| **领用** | **按钮** | 触发自动化 |

约定：日常不直接改「现存量」，只填待领数量后点按钮。

### 1.2 按钮 + 自动化建议

1. 站库存 → 加字段 → 类型选 **按钮**，标题 `领用`
2. 执行操作 → **执行自动化** → 新建
3. 建议开启 **确认弹窗**（如：确认日常消耗出库？）
4. 步骤示例：

```
触发：点击按钮「领用」时（当前记录）

① 条件：待领数量 > 0 且 待领数量 ≤ 现存量
② 新增记录 →「出库流水」
     站点/名称/规格/单位/货位/物料编码 ← 当前站库存行
     数量 ← 待领数量
     出库类型 ← 消耗（默认）
     领用人 ← 待领用人 或 触发人
     台账行 ← 关联当前记录
③ 修改当前站库存
     现存量 ← 现存量 - 待领数量
     待领数量 ← 空/0
④ （可选）写入「修改记录」审计行
```

若自动化不支持「现存量 - 待领数量」，可改为：按钮打开出库表单；另建「出库流水新增时」再扣关联台账行。

### 1.3 单件验收

1. 填 `待领数量=1`，点「领用」→ 确认  
2. 出库流水多 1 行；现存量 -1；待领数量清空  

---

## 2. 插件批量领用

1. 打开 **站库存**，勾选多行  
2. 边栏插件 → **批量领用** → **读取勾选**  
3. 改数量、领用人、类型（默认「消耗」）  
4. 核对 → 确认过账：写出库流水 → 重读账面校验 → 扣现存量 → 写修改记录  

数量保护：过账前重读现存量；账面已变则拒绝；多行顺序过账，失败即停。

---

## 3. 安全约定（公开仓库）

本仓库**不得**包含：

- Personal Base Token / App Secret / 密码  
- 具体 Base 链接、app_token、table_id  
- 内网 IP、私有化域名、员工账号  

插件只通过 `@lark-base-open/js-sdk` 操作**当前已打开的 Base**，不内置任何远端凭证。

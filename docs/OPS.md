## 0. 职责拆分（先读这段）

| 场景 | 用什么 | 不做什么 |
|------|--------|----------|
| **单行日常领用/消耗**（约 90%） | **海能「站库存」表上的原生「领用」按钮 + 自动化** | 不要靠插件行内小按钮当主入口 |
| **多行一次领用**（不同规格/数量） | **边栏插件「批量领用」**：表内勾选 → 读取勾选 → 二次确认过账 | 不要在插件里仿造整张台账当主操作面 |
| 直接改「现存量」 | 仅盘点/纠错（并写修改记录） | 禁止改数后系统猜「是不是领用」 |

对齐先例：物资管理插件是「在子表勾选 + 助手里批量验收/入库」；单行快捷动作应尽量用多维表格按钮/自动化。

插件地址（强刷后应见 ≥ v0.3）：

- https://feishu-plugins.pages.dev/station-material-admin/
- 备用：https://production.feishu-plugins.pages.dev/station-material-admin/

若仍显示 `v0.1.0 · …03:48`，说明服务地址指到了旧部署：在插件设置里改 URL，或 Ctrl+F5。

## 1. 原生「领用」按钮（你在海能 UI 配置）

按钮字段 **不能**用 OpenAPI 可靠创建（type=3001 只读），须在海能里点「+」加列。

### 1.1 推荐字段（站库存）

| 字段 | 类型 | 用途 |
|------|------|------|
| 待领数量 | 数字 | 点按钮前填本次数量；过账后清零 |
| 待领用人 | 文本 | 可选；写入出库流水「领用人」 |
| **领用** | **按钮** | 触发自动化（见下） |

「现存量」建议对站员保持可编辑但约定：**日常不改**；只填待领数量后点按钮。

### 1.2 按钮配置要点

1. 站库存 → 加字段 → 类型选 **按钮**，标题 `领用`
2. 执行操作 → **执行自动化** → 新建
3. 勾选 **开启「确认弹窗」**（标题如：确认日常消耗出库？）
4. 自动化步骤建议顺序：

```
触发：点击按钮「领用」时（当前记录）

① 条件：待领数量 > 0 且 待领数量 ≤ 现存量
② 新增记录 →「出库流水」
     流水号 = "CK-" + 时间戳（或自动编号）
     站点/名称/规格/单位/货位/物料编码 ← 当前站库存行
     数量 ← 待领数量
     出库类型 ← 消耗（默认）
     领用人 ← 待领用人 或 触发人
     台账行 ← 关联当前记录
③ 修改记录 → 当前站库存
     现存量 ← 现存量 - 待领数量
     待领数量 ← 空/0
     最近变动日 ← 今天
④ （可选）新增「修改记录」审计行：操作类型=出库，旧值/新值
```

若海能自动化不支持「现存量 - 待领数量」公式引用，可改为：按钮只 **打开出库表单**（预填关联台账行），再另建自动化「当出库流水新增且类型=消耗/领用时」去扣减关联台账行——以你环境实际能力为准。

### 1.3 验收（单件）

1. 选一行耗材，填 `待领数量=1`，点「领用」→ 确认  
2. 出库流水多 1 行；现存量 -1；待领数量清空

## 2. 插件批量领用（对齐物资「勾选验收」）

适用：一次领多种规格、或要二次核对数量清单。

1. 打开 **站库存** 表，勾选多行  
2. 打开边栏插件 → **批量领用**  
3. 点 **读取勾选** → 改各行数量、领用人、类型（默认「消耗」）  
4. 「核对」→「确认过账」：写出库流水 → 重读账面校验 → 扣现存量 → 写修改记录  

数量保护：过账前重读现存量；与点选时不一致则拒绝；多行顺序过账，失败即停。

## 3. 部署更新（维护者）

```powershell
cd scripts/feishu_bitable/plugin/station-material-admin
npm run build
cd ..
Remove-Item -Recurse -Force pages-dist\station-material-admin -ErrorAction SilentlyContinue
Copy-Item -Recurse station-material-admin\dist pages-dist\station-material-admin
# 主域名与 production 别名都要更，否则 feishu-plugins.pages.dev 会停在旧包
npx wrangler pages deploy pages-dist --project-name=feishu-plugins --branch=main --commit-dirty=true
npx wrangler pages deploy pages-dist --project-name=feishu-plugins --branch=production --commit-dirty=true
```
